# Инструкция: Интеграция модуля СМС-уведомлений (SMS-Gateway v2.1)

Данное руководство предназначено для внутренних команд разработки и тестирования. Документ описывает процесс подключения и настройки модуля СМС-оповещений для сервисов Личного Кабинета.

---

## 🚀 Быстрый старт

Для успешного подключения модуля необходимо последовательно выполнить следующие шаги:

1. Проверить доступность сетевого эндпоинта шлюза.
2. Зарегистрировать уникальный идентификатор `Service_ID` в системе логирования.
3. Настроить конфигурационный файл и составить шаблон СМС.

---

## 🛠️ Параметры конфигурации

Все настройки интеграции передаются в систему через конфигурационный файл. Ниже приведены основные параметры, которые необходимо заполнить:

| Параметр | Тип данных | Описание | Обязательный |
| :--- | :--- | :--- | :--- |
| `gateway_url` | String | URL-адрес шлюза отправки сообщений | **Да** |
| `timeout_ms` | Integer | Время ожидания ответа от сервера (по дефолту `5000`) | Нет |
| `retry_count` | Integer | Количество повторных попыток при сбое отправки | **Да** |
| `mask_phone` | Boolean | Автоматическое маскирование номера в логах | Нет |

---

## 🛑 Типичные ошибки авторизации и их решение

Если в процессе интеграции система возвращает ошибку, сверьтесь со списком ниже для оперативного исправления:

* **Ошибка `401 Unauthorized`** Возникает, если передан неверный или просроченный токен авторизации.  
  *Решение:* Обновите значение переменной `auth_token` в личном кабинете разработчика.

* **Ошибка `422 Validation Error`** Система отклонила запрос из-за неверного формата данных (например, в номере телефона указано меньше 11 цифр).  
  *Решение:* Проверьте валидацию полей на фронтенде перед отправкой запроса на бэкенд.

* **Ошибка `504 Gateway Timeout`** Удаленный сервер ведомства не ответил за отведенное время.  
  *Решение:* Если ошибка повторяется регулярно, увеличьте значение параметра `timeout_ms` до `10000`.

---
